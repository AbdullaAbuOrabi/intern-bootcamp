# Contributing Guide

This document explains the workflow for contributing to this internship bootcamp repository.

## Branch-Based Workflow

Work should be done on a separate branch before merging into the main branch.

Example branch names:

- `feature/setup`
- `feature/sdlc-notes`
- `feature/sql-practice`
- `fix/readme-update`

## Workflow Steps

1. Create or switch to a new branch.
2. Make changes in the correct folder.
3. Test the changes.
4. Commit the changes with a clear message.
5. Push the branch to GitHub.
6. Open a pull request.
7. Review the changes.
8. Merge into `main` after approval.

## Commit Message Examples

Good commit messages:

- `Add SDLC summary notes`
- `Add pull request checklist`
- `Update setup documentation`
- `Generate synthetic dataset`

Bad commit messages:

- `update`
- `fix`
- `changes`

## Rules

- Do not push secret keys or passwords.
- Keep `.env` files local.
- Use `.env.example` only for placeholder variables.
- Keep folder names organized.
- Write clear documentation for each task.
