# Pull Request Checklist

Before merging a pull request, check the following:

## Code Quality

- [ ] Code runs without errors.
- [ ] Code is readable and organized.
- [ ] File names are clear.
- [ ] No unnecessary files are included.

## Documentation

- [ ] README files are updated if needed.
- [ ] Comments are added where helpful.
- [ ] The purpose of the change is clear.

## Testing

- [ ] Python scripts were tested.
- [ ] Database connections were tested if related.
- [ ] Generated files are saved in the correct folders.

## Git and GitHub

- [ ] Changes are committed with a clear commit message.
- [ ] No secret files are pushed.
- [ ] `.env` is not uploaded.
- [ ] `.env.example` is safe and does not contain real keys.
