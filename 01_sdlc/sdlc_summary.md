# SDLC Summary

SDLC stands for Software Development Life Cycle. It is the process used to plan, build, test, deploy, and maintain software.

## 1. Requirement Gathering

In this phase, the team understands what the users or business need.

Example:
For an online store, the team asks what features are needed, such as user login, product listing, shopping cart, payment, and order tracking.

## 2. Design

In this phase, the team plans how the system will work before writing code.

Example:
For an online store, the team designs the database tables for customers, products, orders, and payments. They also plan the app screens and system architecture.

## 3. Development

In this phase, developers write the code.

Example:
Developers build the login page, product page, checkout page, and backend APIs.

## 4. Testing

In this phase, the team checks if the system works correctly and finds bugs.

Example:
The team tests if users can add products to the cart, complete payment, and receive order confirmation.

## 5. Deployment

In this phase, the application is released so users can access it.

Example:
The online store is deployed to a cloud server and made available to customers.

## 6. Maintenance

In this phase, the team fixes issues, improves performance, and adds new features after release.

Example:
The team fixes payment bugs, improves page speed, and adds new discount features.

## Agile and Scrum

Agile is a flexible way of building software in small steps instead of building everything at once.

Scrum is a common Agile framework. It uses short work periods called sprints.

Important Scrum terms:

- User Story: A simple description of what the user needs.
- Sprint: A short period of work, usually 1 to 2 weeks.
- Daily Standup: A short meeting to discuss progress and blockers.
- Retrospective: A meeting after the sprint to discuss what went well and what can improve.

## Git Basics

Git is used to track changes in code.

Important Git concepts:

- Commit: Saves a version of your work.
- Branch: A separate workspace for making changes safely.
- Merge: Combines changes from one branch into another.
- Pull Request: A request to review and merge changes.
- Tag: A label for a specific version of the project.

For this internship bootcamp project, the SDLC can be applied like this:

- Requirement Gathering: Understand bootcamp tasks and deliverables.
- Design: Create repository structure and plan folders.
- Development: Write scripts, generate data, and create documentation.
- Testing: Test Python libraries, PostgreSQL, MongoDB, and generated files.
